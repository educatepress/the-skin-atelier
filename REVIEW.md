# AI 検閲（レビュー）ルール集

このファイルは、コンテンツやコードをレビューする際に **毎回チェックすべき観点** をまとめたものです。
新しいコンテンツを生成・投稿する前、またはコードを変更する前に、このファイルを読んでください。

---

## コンテンツ検閲（ブログ・X・Instagram 共通）

### 医療コンテンツの安全性（YMRL — 健康に関わる重要情報）

美容皮膚科の情報は読者の健康に直結するため、最も厳しい基準で検閲します。

- [ ] **エビデンスレベル:** Tier A/B（ヒト臨床試験・ガイドライン）のみ引用しているか？ Tier C（動物実験・試験管実験）を根拠にしていないか？
- [ ] **断定表現の回避:** 「必ず治る」「絶対に効く」「No.1」等の断定・最上級表現がないか？
- [ ] **恐怖煽り禁止:** 不安を煽って行動させる構成になっていないか？ Empowerment 型（前向きな選択肢の提示）になっているか？
- [ ] **架空エピソード禁止:** 「今日の診察でこんな患者さんが…」等の架空の患者エピソードがないか？ これは **最重要禁止事項**
- [ ] **論文引用の正確性:** 参考文献は実在する論文か？ PMID・著者名・雑誌名が正確か？ 存在しない論文を捏造していないか？

### 禁止ワード・表現

投稿前に必ず以下の語句が含まれていないか検索すること。

**絶対禁止:**
- 「白金高輪」「広尾」「当院」「開業」「オープン」（クリニック所在地の示唆）
- 「分子栄養学」「オーソモレキュラー」（使用禁止の用語）
- 具体的な価格・料金・キャンペーン情報
- 「絶対」「最高」「No.1」（誇大表現）
- 「フォトフェイシャル」（→ 「IPL治療（光治療）」と書く）

**品格に欠ける比喩（ビューティー・マナー違反）:**
- 「錆びる」「サビ」→ 「酸化ストレス」「エイジングの加速」
- 「焦げる」「コゲ」→ 「糖化反応」「透明感の停滞」
- 「腐る」「腐敗」→ 「ターンオーバーの乱れ」
- 「老廃物の塊」「毒素」→ 「代謝産物」「循環の停滞」
- 「カタツムリ」「蛇毒」→ 成分名（ムチン、ペプチド等）で表記

### トーン＆ペルソナの一貫性

- [ ] **Dr. Miyaka のペルソナ:** 「白衣を着た完璧な医者」ではなく「一人の女性として悩みに寄り添う伴走者」の温度感か？
- [ ] **文体:** 40代の洗練された女性が親しい友人に宛てた手紙のような、優しく品のある日本語か？
- [ ] **パラグラフ:** スマホで読みやすいよう、1段落3文以内に収まっているか？
- [ ] **自己開示のバランス:** 医師としての権威を保ちつつ、人間味のある自己開示が適度にあるか？

### プラットフォーム別の追加チェック

**X (Twitter):**
- [ ] ツイート全体が 120 文字以内（全角換算）か？（Twitter Free Tier の 280 weight 制限）
- [ ] ハッシュタグは 3-5 個で、hashtag-strategy.md のバンクから選ばれているか？
- [ ] 論文引用は APA 省略形（著者名 et al., 雑誌略称, 年）か？ PMID の数字列だけ載せていないか？

**ブログ:**
- [ ] 記事末尾に Dr. Miyaka のサイン画像 (`![Dr. Miyaka Signature](/images/miyaka-signature-trimmed.png)`) があるか？
- [ ] 参考文献セクションに実在の論文が PMID 付きでリストされているか？（曖昧な一文で代用していないか）
- [ ] 見出しに番号（1. 2.）や英語が入っていないか？

**Instagram:**
- [ ] ハッシュタグは 15 個（Big 2 + Medium 4 + Niche 6 + Seasonal 3）の構成か？
- [ ] キャプション冒頭 125 文字以内にフック（注意を引く一文）があるか？（125 文字で「...もっと見る」に折りたたまれるため）
- [ ] CTA（フォロー誘導・保存促進）が含まれているか？

## コード検閲

### データ整合性（最重要）

- [ ] **HEADERS 列順:** `sheets.ts` と `sheets-rest.ts` の HEADERS 定義が完全一致しているか？（過去に列ズレ事故あり）
- [ ] **ステータスフロー:** pending → approved → posted の遷移が正しいか？ 別の列に書き込んでいないか？
- [ ] **日付ロジック:** `scheduled_date` は翌日 JST を設定しているか？ 「今日」と「明日」のズレがないか？

### API 呼び出し

- [ ] **Gemini API:** `withRetry()` で指数バックオフ retry が入っているか？ 503 対策なしで直接呼んでいないか？
- [ ] **Twitter API:** 280 weight 制限を超える投稿をしていないか？
- [ ] **Google Sheets API:** サービスアカウント認証を使っているか？ OAuth を使っていないか？

### 環境変数

- [ ] **Make.com webhook:** `MAKE_PUBLISH_WEBHOOK_URL` と `MAKE_IG_PUBLISH_WEBHOOK_URL` の両方を fallback チェーンで扱っているか？
- [ ] **秘密情報:** `.env.local` の値がコードにハードコードされていないか？ コミットに含まれていないか？

### プロンプト保護

- [ ] **prompts/ 配下のファイル** を変更していないか？（医師監修済みの核資産。変更には取締役承認が必要）
- [ ] ハッシュタグ戦略 (`hashtag-strategy.md`) は運用ファイルなので変更OK — ただし構造（Big/Medium/Niche/Seasonal の4層）を壊していないか？

## 検閲の実施手順

1. このファイル (REVIEW.md) を最初に読む
2. 対象のコンテンツまたはコードを確認する
3. 上記チェックリストを1項目ずつ確認する
4. 問題を発見したら ISSUES.md に記録し、修正案を提示する
5. 新たな検閲観点を発見したら、このファイルへの追加を提案する

---

# 運用ルール

- 検閲中に新たなチェック観点を発見したら、このファイルへの追加を取締役に提案する
- 誤検知が多い項目は条件を調整する（厳しすぎ/緩すぎの両方を避ける）
- プロンプト（`prompts/` 配下）の検閲基準が変わった場合は、このファイルも同期する
