```
[毎日 JST 04:00] GitHub Actions (auto-daily-content.yml)
  → scripts/auto-post-blog.ts
    1. ThemeScheduleから翌日テーマ取得 (なければ seasonal-themes で自動生成)
    2. Gemini Deep Research (Google Search Grounding + PubMed PMID確証)
    3. Gemini 記事生成 (blog-writing-guide.md + publishDate明示)

    6. Slack #skin-atelier_jp に承認メッセージ送信
    7. 続いて X 投稿生成 → 同様に承認待ち

[毎日 JST 12:00 / 18:00 / 21:30] GitHub Actions (x-variety-daily.yml)
  → scripts/auto-publish/x-variety-single.ts
  → パターンごとに単発ツイート生成 (🔥myth / 📚evidence / ⚖️contrast / 💌confession / 💭qa)

  ⚠️ 現在 @SatoT1201 credentials になっているため @dr_miyaka_skin に投稿できない (要対応)

[Slack で✅承認クリック]
  → https://skin-atelier.jp/api/slack/interactive
  → Next.js after() でバックグラウンド処理
    1. Sheets status='approved' + scheduled_date=翌日JST
    2. 元メッセージを「✅承認済み」に書き換え

[毎日 JST 10:00] Vercel Cron (/api/cron/daily-publisher)
  → approved かつ scheduled_date=today のアイテムを取得
  → blog: GitHub に `.md` push → Vercel自動デプロイ → サイト公開
  → x: Twitter API で投稿
  → status='posted', post_url=... に更新
```

## 🧠 重要な技術的教訓 (次セッションで忘れないこと)

### 1. Gemini の PMID幻覚問題
- Gemini は「それらしい8桁数字」を PMID として捏造する
- **必ず PubMed MCP で実在確認** してから引用
- パイプラインのプロンプトに「捏造絶対禁止」と明記済み

### 2. Next.js 16 の仕様変更
- `middleware.ts` → `proxy.ts` にリネーム (両方共存不可)
- `after()` from `next/server` が安定版 (バックグラウンド処理)

### 3. Slack 3秒ルール
- Interactive Request URL のレスポンスは 3秒以内
- 重い処理は `after()` で逃がす
- 重いモジュールは **lazy import** (cold start対策)

### 4. Twitter API の制約
- credentials は **投稿先アカウント1つにバインド**
- @SatoT1201 の credentials で @dr_miyaka_skin に投稿不可
- Free Tier で月1500ツイート可能 (自動化に十分)

### 5. Slack Bot チャンネル招待
- 新Botは `/invite @botname` でチャンネル招待しないと `not_in_channel` エラー

### 6. 独立環境の罠
- webpage.new (book) と the-skin-atelier は**完全に独立したプロジェクト**
- Slack App も別 (Reels Factory Bot vs skin atelier Bot)
- Request URL が別ドメインを指していると承認が動かない
