### ⑤ Vercel Cron → GitHub Actions 移植 【着手中・ここが次の作業】

**背景**:
- Vercel Hobby の cron は "best-effort"。過去の posted_at を見ると最大 4.5時間の遅延:
  - 4/17 scheduled → posted 14:35 JST (+4.5h)
  - 4/18 scheduled → posted 10:46 JST (+46min)
- daily-publisher (10:00 JST 予定) が遅延すると、投稿時刻がズレて analytics も劣化

**移植対象** (Vercel `vercel.json` の 2 本):
- `/api/cron/daily-publisher` (JST 10:00)
- `/api/cron/pre-patrol` (JST 15:15)

**実装方針**:
1. 新規 workflow `.github/workflows/daily-publisher.yml` を作成
   - cron: `0 1 * * *` (= 10:00 JST)
   - Vercel endpoint に Bearer CRON_SECRET で POST
   - または endpoint を叩かずに sheet を直接読んで投稿処理（こちらの方が確実）
2. 同様に `.github/workflows/pre-patrol.yml`
3. `vercel.json` から crons を削除 (二重実行防止)
4. エラー時の Slack 通知を既存パターン踏襲

**実装パターン案**: 直接 endpoint POST 方式 (最小差分)
```yaml
name: Daily Publisher (Atelier)
on:
  schedule:
    - cron: '0 1 * * *'  # 10:00 JST
  workflow_dispatch:
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Vercel endpoint
        env:
          CRON_SECRET: ${{ secrets.CRON_SECRET }}
        run: |
          curl -sS -f -H "Authorization: Bearer $CRON_SECRET" \
            https://skin-atelier.jp/api/cron/daily-publisher
```
→ Vercel は引き続き実行ホスト、GitHub Actions はトリガーのみ。**最小変更で確実**。

**注意点**:
- `CRON_SECRET` は既に GitHub Secrets に登録済み（要確認）
- vercel.json から該当 cron を削除しないと二重実行
